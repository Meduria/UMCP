§ I Problem Statement

AI systems are stateless by design.
Context is lost at every session boundary.

01

Stateless by Design

Every AI session initialises from zero. Practitioners re-establish the same context, decisions, and preferences at each interaction — a significant and compounding inefficiency.

02

No Cross-Tool Memory

Separate AI tools operate in isolation. Context accumulated in one environment does not transfer to another. Each tool remains an information island with no formal exchange mechanism.

03

Absence of a Standard

No open specification exists for AI memory interchange. Each tool implements proprietary, incompatible approaches — preventing the emergence of a shared, interoperable memory layer.

§ II Protocol Specification

Three operations.
One unified interface.

UMCP defines a minimal, transport-agnostic API surface. Any AI tool can implement the full specification in an afternoon.

Method Endpoint Description
POST /memory Store a new memory node with type, content, and optional graph relations.
POST /context Query relevant memory for a task. Returns nodes ranked by hybrid score.
PATCH /memory/:id Update an existing node. Timestamps and graph edges are preserved. 
// A single interface across all tools
const client = new UMCPClient({
  storage: new JsonFileStorage()
});

// Claude stores an architectural decision
await client.store({
  type:    "decision",
  content: "Adopted PostgreSQL for
           relational queries"
});

// Cursor retrieves context hours later
const ctx = await client.query({
  task:  "build authentication system",
  limit: 5
});

// → "Adopted PostgreSQL..." ✓
§ III Retrieval Algorithm

Context ranked by
relevance, recency, and relation.

The scoring function combines three independent signals. Weights α, β, γ are configurable per deployment context.

Score(n) = α ·S  +  β ·R  +  γ ·G
S

Semantic Similarity

Jaccard similarity between query tokens and node content. Surfaces the most contextually relevant memories for a given task.

R

Recency Decay

Exponential decay applied with a 7-day half-life. Recent context ranks higher; outdated decisions recede naturally over time.

G

Graph Proximity

Nodes related to other high-scoring nodes receive elevated rank. Retrieval follows the structural topology of the project.

§ IV Storage Backends

Two backends.
One interface.

Both backends implement the same storage contract. Migration between them requires changing a single constructor argument.

Backend v1

JSON File

Zero dependencies. Human-readable. Local-first.

  • No installation required
  • Files remain human-readable
  • Compatible with version control
  • Fully offline, no external services
  • Full file scan on every query
  • Not recommended beyond 10 000 nodes
Recommended for: personal projects, CLI tools, prototypes
Backend v2

SQLite

Production-grade. Indexed. Still local-first.

  • Fast indexed query execution
  • Scales to thousands of nodes
  • Filtering by type and date range
  • Full ACID transaction guarantees
  • Requires better-sqlite3 dependency
  • Marginally more configuration
Recommended for: production deployments, multi-tool environments
